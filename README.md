# 养生小白AI分身

> 实时状态驱动的"养生拓麻歌子"——通过多专家 Agent 协同博弈，将枯燥的健康记录转化为具有情感羁绊的生命养成。

## 项目定位

HealSync 解决传统养生 App 的四大痛点：

| 痛点            | 解决方案               |
| ------------- | ------------------ |
| 交互被动（问答式）     | Multi-Agent 主动干预   |
| 建议同质化（不分场景）   | 基于体质+情境+历史的精准决策    |
| 反馈延迟（看不见长期效果） | 分身 FSM 状态机实时演化     |
| 记录无成就感        | 养生指数 + 健康日历 + 情感羁绊 |

## 核心功能模块

### 1. 数字化身 FSM（有限状态机）

**双轨驱动机制**：

| 维度  | 时间跨度   | 输入          | 输出效果            |
| --- | ------ | ----------- | --------------- |
| 即时态 | 单次交互   | 当前餐食 + 即时情境 | 表情、颜色、即时反馈      |
| 演化态 | 7天滚动窗口 | 平均养生分       | 外观渐变、动作敏捷度、装扮解锁 |

### 2. 双维评价体系

```
┌─────────────────────────────────────────────────┐
│                  饮食记录输入                     │
│         菜品 + 餐厅信息 + 情境标签                │
└─────────────────────┬───────────────────────────┘
                      │
          ┌───────────┴───────────┐
          ▼                       ▼
┌──────────────────┐    ┌──────────────────┐
│  即时养生指数     │    │   营养契合度      │
│  (Instant Wellness)│    │(Nutritional Alignment)│
│                  │    │                  │
│ 评估单餐对当前   │    │ 评估宏量元素是否  │
│ 状态的补益程度   │    │ 符合长期目标      │
│                  │    │                  │
│ 短期视角         │    │ 长期视角          │
└──────────────────┘    └──────────────────┘
```

### 3. 健康账本（动态日历）

- **视觉看板**：类 GitHub 贡献图的色块日历，深浅代表每日综合健康得分
- **复盘功能**：点击日期穿透查看当天的 Agent 评价流与饮食照片

### 4. 多专家协同决策中枢

#### Agent 角色矩阵

| Agent | 角色定位 | 核心立场 | 搜索能力 |
| ----- | -------- | -------- | -------- |
| **老中医** | 养生专家，精通《黄帝内经》 | 节气、性味、体质匹配，守护健康底线 | ✅ 有 |
| **营养师** | 临床营养学博士，数据至上主义者 | 热量、GI值、血糖平稳、营养缺口分析 | ✅ 有 |
| **情绪疗愈师** | 资深心理咨询师 | 情感协调，寻找各方冲突的平衡点 | ❌ 无 |
| **快乐分身** | 用户内心的享乐主义者 | "活在当下"，追求即时多巴胺奖励 | ❌ 无 |
| **自律分身** | 用户未来的成功版本 | "延迟满足"，坚持长远健康目标 | ❌ 无 |
| **老己** | 合一自我，最终决策者 | 整合冲突建议，输出明确行动指令 | ❌ 无 |

#### Agent 博弈机制（两轮辩论）

```
┌────────────────────────────────────────────────────────────────────────┐
│                          两轮辩论流程                                    │
│                                                                        │
│   输入：用户情境 + 饮食意图                                              │
│              │                                                         │
│              ▼                                                         │
│   ┌──────────────────────────────────────────────────────┐            │
│   │         第一轮：独立发言 (Independent Thinking)        │            │
│   │                                                      │            │
│   │  五个Agent基于初始输入独立发言，互不干扰               │            │
│   │                                                      │            │
│   │  ┌─────────┐  ┌─────────┐  ┌─────────┐              │            │
│   │  │ 老中医   │  │ 营养师   │  │情绪疗愈师│              │            │
│   │  │ +搜索   │  │ +搜索   │  │         │              │            │
│   │  └─────────┘  └─────────┘  └─────────┘              │            │
│   │  ┌─────────┐  ┌─────────┐                           │            │
│   │  │快乐分身 │  │ 自律分身 │                           │            │
│   │  └─────────┘  └─────────┘                           │            │
│   └──────────────────────────────────────────────────────┘            │
│              │                                                         │
│              ▼                                                         │
│   ┌──────────────────────────────────────────────────────┐            │
│   │         第二轮：辩论回应 (Selective Interaction)       │            │
│   │                                                      │            │
│   │  每个Agent获得上一轮所有人的发言日志                   │            │
│   │  选择1-2个最支持或最反对的观点进行回应                 │            │
│   │                                                      │            │
│   │  【冲突响应】快乐分身 vs 自律分身 → 情绪疗愈师调停     │            │
│   │  【知识转化】营养师数据 → 老中医中医概念对应           │            │
│   └──────────────────────────────────────────────────────┘            │
│              │                                                         │
│              ▼                                                         │
│   ┌──────────────────────────────────────────────────────┐            │
│   │              最终决策：老己归纳总结                     │            │
│   │                                                      │            │
│   │           ┌─────────────────────┐                   │            │
│   │           │   老己 (合一自我)    │                   │            │
│   │           │ 监听所有论点         │                   │            │
│   │           │ 权衡最大公约数       │                   │            │
│   │           │ 输出JSON格式决策     │                   │            │
│   │           └─────────────────────┘                   │            │
│   └──────────────────────────────────────────────────────┘            │
│              │                                                         │
│              ▼                                                         │
│   输出：博弈结果 + 今日宜忌 + 晚餐推荐 + 行动指数 + 分身状态            │
│                                                                        │
└────────────────────────────────────────────────────────────────────────┘
```

#### 输出格式示例

```json
{
  "博弈结果": "各方达成共识，建议适度满足但控制分量",
  "核心心态调整": "接纳当下的欲望，但为明天留余地",

  "行动指数": 7,
  "分身状态": "happy"

```

### 5. 交互节律

**固定同步**：

- 晨起：睡眠/心情同步
- 午间：膳食/环境同步
- 睡前：日历复盘

**即时干预**：

- 救赎模式：吃多了/熬夜了
- 身体警报：胃胀/偏头痛

```
             【行动指数】7

T2 (反馈与闭环)
├── 即时：获得"温柔自律"勋章，分身状态稳定
```


### 数据持久化说明

ModelScope 创空间中，`/mnt/workspace` 是持久化存储目录，应用重启后数据不会丢失：

| 路径                       | 用途     | 说明                |
| ------------------------ | ------ | ----------------- |
| `/mnt/workspace/data/`   | 用户数据存储 | 用户画像、历史决策、养生指数等   |
| `/mnt/workspace/logs/`   | 运行日志   | 应用运行日志、Agent 决策日志 |
| `/mnt/workspace/config/` | 配置文件   | 可选的动态配置           |

### 部署步骤

#### 1. 准备代码仓库

确保项目包含以下文件：

- `frontend/Dockerfile` - 前端构建配置
- `backend/Dockerfile` - 后端构建配置
- `docker-compose.yml` - 容器编排配置

#### 2. ModelScope 创空间配置

1. 创建新的创空间，选择 **Docker** 部署方式
2. 连接代码仓库（GitHub/GitLab/Gitee）
3. 设置端口映射：`7860`（前端）
4. 配置环境变量

#### 3. 环境变量设置示例

```bash
# 在 ModelScope 创空间设置页面添加
MODELSCOPE_API_KEY=ms-your-api-key-here
DATA_DIR=/mnt/workspace/data
```
